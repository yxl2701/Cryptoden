# Security Policy

Cryptoden is intended for CTF practice, cryptography learning, security education, and authorized analysis.

## Reporting Vulnerabilities

If you find a security issue, please open a GitHub issue with a minimal reproduction when public disclosure is safe. If private reporting is needed, use the repository owner's preferred contact method once published.

Please include:

- Affected version or commit.
- Reproduction steps.
- Expected and actual behavior.
- Potential impact.

## Sensitive Data

Do not publish API keys, tokens, private keys, local logs, decrypted records, `.env` files, or other secrets. The default repository configuration excludes common runtime and local-secret files.

## Responsible Use

Use this project only where you have authorization. You are responsible for complying with applicable laws, platform rules, competition rules, and authorization boundaries.
